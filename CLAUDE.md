# CLAUDE.md — 成医通项目编码准则

## 项目概况

成医通（成都医学院校园百科平台），基于 VitePress 构建的文档站点。Markdown 内容驱动，TypeScript 配置文件，Vue 3 默认主题。

## 技术栈

- **框架**: VitePress 2.0 alpha（Vue 3 + Vite）
- **语言**: TypeScript（配置文件）、Markdown（文档内容）
- **样式**: VitePress 默认主题 + 自定义 CSS

## 优先使用 VitePress 自带功能

动手写代码之前，先确认 VitePress 是否已经内置。

**原则：能配置解决的，不写代码。配置不够用的，优先扩展默认主题（`.vitepress/theme/`），而不是另起炉灶。**

## 行为准则

**权衡：谨慎优先于速度。琐碎任务自行判断。**

### 1. 编码前先思考

- 明确陈述假设。有疑问提出，不要掩饰困惑。
- 存在多种解释时全部列出，不默默做选择。
- 有更简单的方法时提出来，必要时要坚持己见。
- 遇到不清楚的地方停下来，说出困惑点，然后提问。

### 2. 简单至上

- 用最少改动解决问题，不做推测性添加。
- 不添加超出要求的功能。
- 不为一次性内容做抽象。
- 不提供未要求的"灵活性"或"可配置性"。
- 问问自己："一位资深工程师会认为这过于复杂吗？" 是的话就简化。

### 3. 手术式改动

- 只碰必须碰的，只收拾自己的烂摊子。
- 不"改进"相邻的代码、注释或格式。
- 不重构没有损坏的代码。
- 保持与现有风格一致，即使做法不同。
- 发现无关的死代码/死文件时指出来，但不删除。
- 每行修改都应直接追溯到用户的请求。

### 4. 目标驱动执行

- 将任务转化为可验证的目标，定义成功标准。
- 多步骤任务先简述计划。
- 验证方式：`npm run docs:dev`（启动开发服务器，浏览器预览确认）或 `npm run docs:build`（构建生产版本，零错误通过才算成功）。
