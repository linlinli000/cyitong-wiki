# 贡献指南

欢迎你参与成医通的编写与完善！本文档将说明如何贡献内容以及需要遵守的规范。

## 参与方式

- **GitHub Pull Request**：直接 Fork 仓库，修改后提交 PR。适用于内容较多或结构性的修改。
- **GitHub Issues**：提出改进建议、报告内容问题或讨论页面结构。
- **交流群**：加入成医通交流群（待补充），与其他贡献者沟通。

---

## 内容立场

1. **客观中立**：成医通是开源的知识共享平台，内容应保持客观、中立、准确。与成都医学院校方无关，不代表校方立场。
2. **引用官方来源**：涉及政策、规定、数据等内容，务必引用学校官方通知、教务处文件等可查证来源。
3. **去主观化**：避免"我觉得""建议""推荐"等主观措辞。如需给出建议，请使用 `:::tip` 容器并注明依据。
4. **维护良好氛围**：不得利用平台制造对立、引战或散布不实信息。

## 署名规范

在 Markdown 文件的 frontmatter 中通过 `author` 字段署名：

```yaml
---
title: 页面标题
author: <GitHub 用户名>  # 或 -qq-<QQ号>
---
```

**规则**：

- **GitHub 用户**：直接填写 GitHub 用户名，系统将自动获取头像和主页链接。
- **QQ 用户**：使用 `-qq-<QQ号>` 格式，如 `-qq-123456789`。可附加显示名称：`-qq-123456789-昵称`。
- **多作者**：使用 YAML 数组格式：

```yaml
author:
  - github-user
  - -qq-123456789-昵称
```

署名会显示在文档内容结束后的作者区域。

---

## 引用与链接政策

1. **优先引用官方来源**：学校官网、教务处通知、学院公告等。官方链接失效时，可补充第三方存档链接。
2. **谨慎引用用户生成内容**：贴吧、知乎、小红书等平台内容作为参考，**不作为事实依据**。使用 `:::warning` 容器标注：
   ```markdown
   :::warning 网络来源
   以下信息整理自贴吧 / 知乎，未经官方确认，仅供参考。
   :::
   ```
3. **外部链接**：使用完整的 HTTPS URL。链接文字应清晰描述目标内容，避免"点击这里"等模糊描述。

## 时效性规则

信息会随时间变化而失效。为保证内容质量，采用三级保留策略：

| 状态 | 处理方式 | 示例 |
|------|----------|------|
| **明确过时** | 使用 `:::danger` 容器标注，说明已知变化 | 宿舍费用调整、报到流程变更 |
| **2 年以上未更新** | 在页面顶部添加 `:::warning 时效提醒`，折叠旧内容 | 课程安排、作息时间 |
| **严重过时（>3 年）** | 经确认后直接删除，在 commit 中记录删除原因 | 已废除的规定、已停用的设施 |

**时效声明示例**：

```markdown
:::warning 时效提醒
本页面内容最后更新于 2024 年 6 月。部分信息可能已过时，欢迎
[协助更新](https://github.com/linlinli000/cyitong-wiki)。
:::
```

## 编辑语法

成医通基于 **VitePress** 构建，内容使用 **Markdown** 编写。常用语法：

### Markdown 基础

```markdown
# 一级标题（页面标题，每页仅一个）
## 二级标题（会出现在右侧目录）
### 三级标题（会出现在右侧目录）

**加粗**、*斜体*、`行内代码`

- 无序列表
1. 有序列表

[链接文字](https://example.com)
![图片描述](./images/example.png)
```

### VitePress 自定义容器

```markdown
:::tip 提示
用于补充说明、小技巧
:::

:::info 信息
用于提供背景信息
:::

:::warning 警告
用于提醒注意事项、标注时效性问题
:::

:::danger 危险
用于标注严重过时内容、重要警示
:::

:::details 详细信息
折叠内容，适合放延伸阅读
:::
```

### 数学公式

成医通已启用 LaTeX 数学公式支持：

```markdown
行内公式：$E = mc^2$

块级公式：
$$
\int_a^b f(x)\,\mathrm{d}x
$$
```

:::tip 医学统计场景
数学公式支持对展示医学统计公式、药代动力学模型等学术内容尤为实用。
:::

## 架构规范

### 目录结构

```
docs/
├── freshman/           # 新生指南
│   ├── index.md
│   ├── preparation.md
│   │   images/         # 页面专用图片
│   │   └── dorm.jpg
│   └── ...
├── study/              # 学习
├── life/               # 生活服务
├── campus/             # 校园
├── index.md            # 首页
├── contributing.md     # 贡献指南（本页）
├── changelog.md        # 更新日志
└── about.md            # 关于我们
```

### 图片与附件

- **图片**：存放在对应页面的 `images/` 子目录中。使用压缩后的格式（WebP 优先，JPG/PNG 其次），确保文件大小合理。
- **附件**：存放在对应页面的 `files/` 子目录中。
- **命名**：使用语义化的英文或拼音命名，如 `campus-map.webp`、`dorm-1.jpg`。避免 `IMG_001.png` 等无意义名称。

### 页面规范

- 每页只有一个一级标题（`#`）。
- 在子目录的 `index.md` 中提供概览和导航。
- 数据驱动页面（专业列表、附属医院等）优先使用 JSON 数据源 + Vue 组件模式，避免在 Markdown 中重复手写结构化信息。
